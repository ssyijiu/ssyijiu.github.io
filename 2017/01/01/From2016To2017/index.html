2016 总结，2017 计划

发表于 2017-01-01 | 分类于 个人总结

前言

先做下自我介绍：Android 程序员一枚，目前就职于国美集团下一家子公司。技术栈以 Android 为主，会点 JavaScript 和 嵌入式开发。

生活

今年五一和女朋友来到了北京，至今已经搬了三次家，但是生活上还算不错，没有为衣食住行发过愁。两个人的感情也从最初的一言不合就争吵，到现在什么事都相互体谅、相互理解。“朝九晚五浪迹天涯” 的心愿，希望早点实现后面四个字。

技术

其实我觉得自己已经很努力了，但还是远远的不够，仍然欠下了很多技术债。

看看 Jacksgong 从最初的在微信工作，到主持英语流利说 Android 项目重构，再到入职支付宝性能架构组。差不多的年纪，比你厉害太多的人太多了。

入职时公司的项目还在使用 Eclipse 开发，各种糟糕的代码让人不能直视，于是这半年在工作之余，一边记录公司项目的各种问题，一边学习各种框架，计划重构。

在技术学习上也主要针对了设计模式和 Android 的项目架构进行学习，同时将目前 Android 流行框架实践了一遍并进行了二次封装。但纸上得来终觉浅，农历新年回来后由我主导的项目重构算是最对这半年学习的一个检验吧。不怕做的不好，就怕失去一根进取的心。

阅读

这是对自己这半年来最不满意的一个地方，买了很到书，但没有一本仔仔细细的读完，大都浅尝辄止。甚至好多书看了个开头，然后一两个月没看，再看的时候甚至要再次从头看起。

不仅书没有好好看，技术文章也被 微博、稀土掘金、微信公众号刷爆了大脑，每天执着于收藏一些文章，这一收藏就是永远不看了。 Later equals never，稍后等于永不，收藏等于放弃。好在在十一月初的时候意识到了这个问题，现在看到一篇文章后会快速从当前或者下一阶段的工作是否需要和是否感兴趣两个方面进行判断，需要的我会分享到微信 “文件传输助手” 上在两天内看完。

英语

第一次意识到英语对一个程序员的重要性是在大学研究 Java8 新特性的时候，各种英文文档读到想吐，这半年来我有意识的去练习英语阅读，麻麻再也不担心我每天刷 Github 什么都看不懂了。

17年我将会持续加强英语阅读，坚持看英文文档，阅读 API Guides，并做一个系列的个人笔记。

游戏

在刚入职现在这家公司的时候，同事问我玩不玩 LOL（>.< 确实不玩），后来和大家玩了一段时间的“部落冲突”，有时候玩游戏确实成了一种职业技能。

PC 端的游戏没有再碰过，算是告别玩了五六年的 DNF（靠，不冲点券想玩个毛） 和大学最喜欢的 “神之浩劫”（电脑带不动了>.<）

至于王者荣耀，差不多是工作日不玩，周末休息的时候会玩。

17年要实现两个目标：

  • 段位打上铂金（ >.< 彩笔一枚，打脸就打脸吧，毕竟游戏）

  • 不影响生活，比如家人叫你的时候立刻放下游戏。

2017

2017年我将会把主要精力放在技术学习上，深入学习 Android，追求最佳实践，踏踏实实看几本书，进行技术栈拓展，开阔思路。

Android 继续深入学习

  • 架构与设计模式

  • 优秀开源项目及 Android 源码阅读（根据兴趣阅读）

  • Android 前沿技术实践，包括插件化、热补丁、组件化、RxJava、Kotlin。吐血推荐 2016移动开发技术巡礼

阅读

  • Android 群英传

  • Android 开发艺术探索

  • Clean Code

  • Effective Java

技术栈拓展

  • Android Things

  • Vue

  • AI

最后：积累自己的代码库，整理笔记。

具体实施计划

  • 农历新年期间完成一套 Android 项目基础框架的封装，并完成 Android群英传神兵利器的阅读（2017/2/6）

  • 接着用 2 个月的时间对 Android 前沿技术进行实践，并完成一个 Gank & Github 的客户端，之后持续对其优化（2017/4/1）

  • 阅读 Android 群英传 ，picasso & retrofit 源码，Square 粉，picasso 算是图片加载库中简单的了，retrofit 又是设计模式的典范，预计一个月时间（2017/5/1）

  • 开始Android 开发艺术探索的初步阅读，时间一个月，读完第一遍后需要继续反复阅读 (2017/6/1)

  • 阅读 Clean Code & Effective Java，提升代码水平，大约两个月时间 (2017/8/1)

  • 进行半年的总结，知识沉淀消化，一个月时间 (2017/9/1)，这个时候，如果前面坚持下来的话，Android 技术应该可以达到中等偏上的水平了。

  • 2017 年 9 月份选择一个方向开始技术栈的拓展。
